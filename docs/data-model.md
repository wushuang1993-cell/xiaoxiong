# 数据模型说明

当前原型里的数据将来会映射到 Supabase：

## 用户和家庭

- `profiles`：每个登录账号的昵称、角色、头像。
- `families`：一个家庭空间。
- `family_members`：闪闪鱼和杰尼龟加入同一个家庭。

## 小熊

- `bears`：小熊目录，包括名字、头像、是否参与。
- `wish_bears`：每个人当前的心愿小熊。

## 规则和积分

- `coin_rules`：规则配置，包含基础家务、增值家务、扣分、饮品。
- `daily_logs`：每天点选的记录。
- `coin_ledger`：金币流水。建议所有金币变化都写入这里，方便回溯。

## 抽签和兑换

- `daily_draws`：每天抽签结果、随机种子、校验码。
- `exchange_requests`：兑换申请、同意状态、对方选择换走哪只小熊。

## 登录后的判断逻辑

用户登录后：

1. 通过 Supabase Auth 得到 `auth.user.id`。
2. 在 `profiles` 找到这个账号是闪闪鱼还是杰尼龟。
3. 在 `family_members` 找到所属家庭。
4. App 读取这个家庭的共同数据。
5. 所有新增记录写入当前登录用户的 `user_id`。

这样两个人看到的是同一个家庭数据，但每条行为都能追踪是谁操作的。
